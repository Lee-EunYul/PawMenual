# PAWMENUAL 개발용 에이전트 지침서 (비개발자용)

이 문서는 "기획서.md"를 실제 웹 서비스로 만들기 위해, AI 코딩 에이전트에게 어떤 방식으로 요청해야 하는지 정리한 실행 가이드입니다.

핵심 목표는 3가지입니다.

1. 한 번에 많이 만들지 않고, 작은 기능부터 안전하게 완성하기
2. 매 단계마다 직접 확인(테스트)하고 다음 단계로 넘어가기
3. 어려운 용어 대신 쉬운 설명으로 진행하기

---

## 1) 프로젝트 목표 한 줄 요약

PAWMENUAL은 강아지 정보를 바탕으로 맞춤형 사료 추천, 건강 이상 상황 확인과 주변 동물병원 안내, 반응형 훈련 방법을 제공하는 초보 보호자용 웹 서비스다.

---

## 2) 에이전트에게 항상 지켜달라고 할 규칙

아래 문장을 매 요청에 붙여서 사용하세요.

- 나는 비개발자다. 초등학생도 이해할 수 있게 쉬운 한국어로 설명해줘.
- 한 번에 하나의 기능만 개발해줘.
- 작업 후 "무엇을 바꿨는지", "왜 바꿨는지", "내가 어떻게 확인하는지"를 3줄로 요약해줘.
- 필요한 터미널 명령어는 복붙 가능한 형태로 한 줄씩 제공해줘.
- 내가 확인하기 전에는 다음 큰 기능으로 넘어가지 마.
- 에러가 나면 원인과 해결 방법을 쉬운 말로 설명해줘.

---

## 3) 개발 방식 (중요)

### 방식: MVP -> 개선

처음부터 완벽한 서비스가 아니라, "동작하는 최소 버전(MVP)"을 먼저 만듭니다.

### 개발 순서

1. 화면 뼈대 만들기
2. 데이터 입력/저장 만들기
3. 기록 조회/수정 만들기
4. 추천/분기 로직 붙이기
5. 디자인 다듬기

### 완료 기준

- 기능이 실제로 클릭되고 저장되어야 완료
- 화면만 있고 저장이 안 되면 완료 아님

---

## 4) 권장 기술 스택 (초보자 친화)

에이전트에게 아래 스택으로 시작하라고 요청하세요.

- 프론트엔드: React + Vite
- 스타일: Tailwind CSS
- 데이터 저장 1단계: localStorage (로그인 없이 빠른 검증)
- 데이터 저장 2단계: Supabase 또는 Firebase (원하면 나중에)

왜 이렇게 하나?

- localStorage로 먼저 만들면 빠르게 동작 확인 가능
- 이후 온라인 DB로 옮기기 쉬움

---

## 5) 기능별 개발 단계 (기획서 기준)

## Step 0. 기본 프로젝트 세팅

목표:

- 프로젝트 실행 성공
- 공통 레이아웃 생성 (상단 제목, 하단 메뉴)
- 기본 페이지 5개 생성

필수 페이지:

- 메인
- 프로필
- 식생활 가이드
- 건강 상황 확인
- 훈련
- 성장 기록

내 확인 방법:

- 페이지 이동이 모두 되는지 클릭 확인
- 모바일 화면에서도 글자가 깨지지 않는지 확인

---

## Step 1. 강아지 프로필 저장 기능

목표:

- 이름, 견종, 나이, 크기, 몸무게, 활동량, 식사 횟수, 음식 선호, 현재 사료 형태 입력
- 저장 후 새로고침해도 값 유지(localStorage)

완료 조건:

- 저장 버튼 누르면 "저장 완료" 메시지 표시
- 이전 값 불러오기 동작

내 확인 방법:

1. 값 입력
2. 저장 클릭
3. 새로고침
4. 값 유지 확인

---

## Step 2. 강아지 맞춤형 사료 추천 + 반응 기록

목표:

- 견종/나이/몸무게/활동량/식습관 기반 추천 카드 제공
- 건식/습식/혼합 특징 비교 + 추천 급여 방식 표시
- 반응 기록(매우 잘 먹음/잘 먹음/보통/잘 먹지 않음) 저장

완료 조건:

- 날짜와 함께 반응 기록이 리스트에 쌓임
- 최근 반응 1개가 메인 화면에 요약 노출

내 확인 방법:

- 서로 다른 반응을 2번 저장해서 목록이 쌓이는지 확인

---

## Step 3. 건강 이상 상황 확인 + 주변 동물병원 안내

목표:

- 상황 선택(노란색 구토/식욕 저하/배변 변화/활동량 감소/기타)
- 세부 체크 항목 선택
- 입력 요약 + 추가 관찰 항목 + 전문가 상담 안내 표시
- 사용자 위치 기반 주변 동물병원 지도/리스트 표시
- 기록 저장

중요 문구(고정):

- 이 기능은 진단이 아니며, 우려 증상이 반복되면 수의사 상담이 필요합니다.

완료 조건:

- 저장한 건강 기록이 성장 기록 화면에서 날짜별 조회 가능
- 위치 허용 시 주변 동물병원 3곳 이상이 지도 또는 리스트로 노출

---

## Step 4. 훈련 시스템 (반응 기반 분기)

목표:

- 훈련 선택(앉기/기다리기/손/엎드리기/돌기 등)
- 단계별 훈련 안내 표시
- 결과 선택:
  - 잘 따라 했어요
  - 일부만 따라 했어요
  - 관심을 보이지 않았어요
  - 방법을 이해하지 못한 것 같아요
- 결과에 따라 다음 단계 제시 또는 다른 방식의 훈련 방법 새롭게 제안

완료 조건:

- 날짜별 훈련 기록 저장
- 어떤 방법에서 반응이 좋았는지 확인 가능

---

## Step 5. 성장 기록 통합 화면

목표:

- 식사 반응 기록, 건강 관찰 기록, 훈련 기록을 한 화면에서 확인
- 날짜 필터 제공

완료 조건:

- 3개 영역 데이터가 모두 보임
- 최신순 정렬

---

## 6) 데이터 구조(초간단 버전)

에이전트에게 아래 구조로 구현 요청하세요.

- profile
  - name, breed, age, size, weight, activityLevel
  - mealCountPerDay, foodPreference, currentFoodType

- mealLogs[]
  - date, foodType, reaction, memo

- healthLogs[]
  - date, category, checks[], summary, cautionLevel, nearbyHospitals[]

- trainingLogs[]
  - date, trainingType, step, result, nextAction

---

## 7) 에이전트에게 복붙해서 쓰는 요청문

## 요청문 A: 단계별 개발 시작

아래 조건으로 Step 0만 구현해줘.

- 나는 비개발자라 쉬운 한국어로 설명해줘.
- 파일을 실제로 수정하고 실행 가능한 상태로 만들어줘.
- 작업 후 내가 확인할 체크리스트 3개를 제시해줘.
- 아직 Step 1 이상은 하지 마.

## 요청문 B: 저장 기능 추가

Step 1(강아지 프로필 저장)을 구현해줘.

- localStorage 기반으로 만들고, 새로고침 후 값 유지가 되게 해줘.
- 저장 성공 메시지를 보여줘.
- 변경한 파일 목록과 역할을 쉬운 말로 설명해줘.

## 요청문 C: 오류 해결

현재 실행 중 에러가 발생했다.

- 원인을 쉬운 말로 2~3가지 가능성으로 설명해줘.
- 바로 고칠 수 있는 순서대로 수정해줘.
- 수정 후 내가 확인할 방법을 알려줘.

## 요청문 D: 배포 전 점검

지금 상태에서 배포 전 점검 체크리스트를 만들어줘.

- 필수 점검, 권장 점검으로 나눠줘.
- 누락된 기능이 있으면 기획서 기준으로 알려줘.

---

## 8) UI/문구 원칙

- 버튼 문구는 짧고 분명하게: 저장하기, 다음 단계, 기록 보기
- 경고 문구는 불안 조장 금지, 행동 유도 중심으로 작성
- 건강 영역은 항상 "진단 아님" 안내 표시
- 폼 입력 에러는 빨간 글씨 + 해결 방법 함께 제공

예시:

- 잘못된 예: 입력값 오류
- 좋은 예: 몸무게는 숫자로 입력해 주세요. 예: 3.5

---

## 9) 내가 매 단계 끝날 때 확인할 것

1. 내가 클릭했을 때 실제 동작하는가?
2. 새로고침 후 데이터가 유지되는가?
3. 문장이 어렵지 않고 보호자 입장에서 이해되는가?
4. 모바일에서도 화면이 깨지지 않는가?

하나라도 아니오면 다음 단계로 넘어가지 않는다.

---

## 10) 이후 확장(선택)

MVP가 끝나면 아래를 순서대로 확장합니다.

1. 로그인 기능 추가
2. 온라인 DB 연동(Supabase/Firebase)
3. 사진 업로드(강아지 프로필 사진)
4. 알림 기능(훈련/급식/관찰 기록 리마인드)
5. 주간 리포트(반응 추세)

---

## 11) 마지막 원칙

- 완벽보다 완성: 작게 만들어서 자주 확인
- 어려운 기술보다 이해 가능한 구조 우선
- 내(기획자) 확인이 끝난 뒤 다음 단계 진행

이 문서대로 진행하면, 비개발자도 AI 에이전트와 함께 PAWMENUAL을 실제 서비스 형태로 완성할 수 있습니다.
